🥶 Freeze Brainrot

Freeze or unfreeze a specific brainrot on the road in whichever server a given player is currently in.

ℹ️ How it works

Looks up the Job ID of whichever server that username is currently in (no need to find it yourself) and sets ForceIdle on all matching brainrots on the road.
Freeze stops them in place. Unfreeze lets them resume movement.
Only affects brainrots currently on the road — not ones in player bases.

🎒 Give Item

Give a tool/item to a player currently in-game — just type their username, no need to find the server yourself.

ℹ️ How it works

Automatically finds whichever server that username is currently in and clones the named tool from ReplicatedStorage.Items into their Backpack.
The player must be currently in-game on that server for this to work.

💨 Force Speed

Force a player's WalkSpeed to a specific value in a target server. Set to 0 to disable.

ℹ️ How it works

Continuously forces the player's Humanoid.WalkSpeed every 0.1s until you set it to 0 to disable.
The player must be currently in-game on that server.
Default Roblox walk speed is 16.

👻 Fake Spawn

Spawn a fake brainrot on the road in whichever server a given player is currently in. When collected it poofs — no reward given. Rolls 15% Gold / 10% Diamond / 5% Rainbow mutation automatically.

⚠️ How it works

1. Enter a Roblox username of anyone currently in the server you want to target — the server is found automatically.
2. Choose a fake type: Fake 1 sets IsFake — the brainrot poofs the moment a player clicks it. Fake 2 sets iffake — it appears to be collected normally and walks to the base, then poofs on arrival. Fake 3 sets FakeInBase — it is collected into the base normally, then poofs once inside.
3. It has a 15% Gold, 10% Diamond, or 5% Rainbow mutation rolled on delivery.
4. Either way it poofs with no brainrot and no reward.
5. Only one fake spawn can be queued per server at a time — queueing another overwrites the previous.

🎯 Remote Spawn

Spawn a real brainrot on the road of whichever server a given player is currently in. The next player who collects it gets it for real — no fake attributes.

ℹ️ How it works

1. Enter a Roblox username of anyone currently in the server you want to target — the server is found automatically.
2. Choose the brainrot name, and optionally a mutation and/or traits.
3. The animal spawns on the road with real attributes — whoever collects it keeps it.
4. Only one remote spawn can be queued per server at a time — queueing another overwrites the previous.

⚡ Remote Event

Trigger a named in-game event in whichever server a given player is currently in.

ℹ️ How it works

1. Enter a Roblox username of anyone currently in the server you want to target — the server is found automatically.
2. Select the event you want to fire from the dropdown.
3. The game server polls every 3 seconds and calls EventService:ExecuteEvent when it receives the queued event.
4. Only one event can be queued per server at a time — queueing another overwrites the previous.

💥 Poof Road

Instantly poof every brainrot currently walking the road in whichever server a given player is in — craft-spawn animals are skipped.

ℹ️ How it works

Looks up whichever server that username is currently in automatically.
On the next poll cycle (≤3s), the game server iterates every model in workspace.MovingAnimals.
Each non-craft-spawn animal gets the Poof VFX fired to all clients, the poof sound played, and is then destroyed.
Animals that originated from the Craft Machine (IsCraftSpawn = true) are skipped and continue walking normally.
This is a one-shot command — queue it again to poof the next wave.

🔊 Play Sound

Play a Roblox sound asset for all clients in whichever server a given player is currently in.

ℹ️ How it works

1. Enter a Roblox username of anyone currently in the server you want to target — the server is found automatically.
2. Enter the Roblox asset ID of the sound you want to play (numbers only — e.g. 84435681380673).
3. On the next poll cycle (≤3s), the game server plays the sound for all clients in that server.
4. Only one sound command can be queued per server at a time — queueing another overwrites the previous.

ℹ️ How it works

1. Create Code — amount is always 1 brainrot per redeem.
2. Triple Code — player receives all 3 chosen brainrots when they redeem the code.
3. The game server polls GET /roblox/redeem-codes to keep its local list in sync.
4. When a player types the code in-game, the game calls POST /roblox/use-redeemcode to record the use.
5. If the code has limited uses, it auto-deletes once fully exhausted.
6. Mutations are rolled by the game at redemption time (Normal 35 · Gold 30 · Diamond 15 · Rainbow 3).
